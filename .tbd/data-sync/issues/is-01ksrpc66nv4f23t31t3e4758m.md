On a multi-worktree setup, '.tbd/config.yml' is per-checkout (branch-visible). When the v0.2.0 client runs its first mutating op from a sibling whose config is still f03, it silently bumps that config to f04 — appearing as an unexpected tracked diff (M .tbd/config.yml) on the user's feature branch.

Add a clear one-time notice on the first such bump, e.g.:

> Note: bumped .tbd/config.yml tbd_format f03 → f04 in this checkout.
>   Commit on this branch or merge main to publish the upgrade.

Observed on ai-trade-arena-disk-cleanup-skill sibling worktree.

Key files:
- packages/tbd/src/file/config.ts (migration write path)
- packages/tbd/src/cli/lib/output-manager.ts (or wherever user-visible notes are emitted)

Tests:
- Add a test under packages/tbd/tests/setup-flows.test.ts or a new sibling-worktree test that asserts the notice is printed on first bump and not on subsequent commands.
